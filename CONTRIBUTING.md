# 贡献指南

感谢您对 XXL-Job Executor MQ Improve 项目的关注！我们欢迎所有形式的贡献。

## 如何参与贡献

1. Fork 本项目
2. 新建分支
   ```bash
   git checkout -b feature/your-feature
   ```
3. 开发与测试
   - 遵循阿里巴巴 Java 开发手册
   - 单元测试覆盖率 > 80%
   - 重要功能需集成测试
4. 提交代码
   ```bash
   git add .
   git commit -m "feat: 新功能描述"
   git push origin feature/your-feature
   ```
5. 创建 Pull Request

## 代码规范

- 4 空格缩进
- 类名 PascalCase，方法名 camelCase
- 常量 UPPER_SNAKE_CASE
- 提交信息遵循 Conventional Commits

## 测试要求

- 单元测试覆盖率 > 80%
- 集成测试覆盖核心功能
- 性能测试验证关键指标

## Bug 报告

请在 Issue 区按模板提交，包含：
- 复现步骤
- 期望行为
- 实际行为
- 环境信息（系统、JDK、MySQL、RocketMQ 版本等）

## 功能建议

请描述清楚场景、需求、优先级
